# Post it 事前検討まとめ

## 1. 現時点の採用方針
- アプリ名は仮称 `sticky`
- 初期MVPは `macOS専用`
- 技術構成の第一候補は `Tauri v2 + React + SQLite`
- クラウド処理は行わない
- メモ、セッション、設定、ゴミ箱はすべてローカル保存
- メモ本文は `プレーンテキスト`
- Markdown装飾やリッチテキストは初期MVPでは扱わない

## 2. プロダクトの芯
- 作業中に思考を止めず、その場で即メモできる
- メモは毎回新規作成感を持つ
- デスクトップ上では他アプリより前面に表示される
- 邪魔ならすぐ閉じられる
- 閉じても管理画面から回収できる
- セッション単位で思考のまとまりを扱う

## 3. セッションモデル
- セッション型を採用する
- セッション開始は `1枚` または `複数枚(2〜9枚)` の2択
- セッション生成後にメモ追加はしない
- 1枚セッションのあとに新規メモを作っても別セッション
- 複数枚セッションで不要メモを削除して1枚になっても、その後の新規メモは別セッション
- セッション移動は管理画面で行う
- セッション移動時は色変更だけでなく `session_id` を移行先へ更新する
- セッション内メモ数が0になったら空セッションは残さず消滅させる

## 4. デスクトップ操作モデル
- 1回目クリックで `セッション選択`
- 2回目クリックで `メモ選択`
- `Enter` または再操作で `編集`
- `Esc` で編集終了または選択解除
- セッション選択中はセッション内全メモを一括移動
- メモ選択中は対象メモだけ移動
- `Cmd + S` で保存
- `Cmd + Enter` で保存して閉じる
- `Delete` で削除確認
- 削除確認中の `Delete` または `Enter` で削除確定

## 5. 表示上限
- 同時展開セッション数は最大 `5`
- 同時表示メモ数は最大 `15`
- 上限到達時は新規作成しない
- デスクトップ上では、既存表示メモ全体に赤半透明オーバーレイと横揺れで警告する
- 管理画面では上限状態を文言表示する

## 6. 初期配置
- メモ表示位置は `15スロット` で管理する
- 配置は `5列 x 3段`
- 画面サイズに対する相対座標で保持する
- 基本座標は以下とする
  - 横: `10 / 30 / 50 / 70 / 90%`
  - 縦: `18 / 43 / 68%`
- 新規表示時は空いている最小番号スロットへ配置する
- ユーザーが移動した場合、その元スロットは再利用可能とする
- 再表示時も同じルールで空きスロットへ配置する

## 7. メモサイズ
- デフォルトサイズ: `320 x 240`
- 最小サイズ: `240 x 180`
- 最大サイズ: `520 x 420`
- `メモ選択中` のときだけ四隅にリサイズハンドルを表示する
- 角ドラッグで拡大縮小可能とする
- アスペクト比固定は行わない
- デスクトップで変更したサイズはその表示中のみ有効
- 管理画面から再展開した場合は毎回デフォルトサイズに戻す

## 8. 画面構成
必要画面は以下とする。

- `デスクトップメモUI`
- `管理画面 Home`
- `管理画面 Trash`
- `管理画面 Settings`

補助UI:
- `削除確認モーダル`
- `上限警告オーバーレイ`
- `初回起動時の空状態UI`

## 9. 管理画面方針
- 管理画面は通常のmacOSアプリウィンドウとして扱う
- 赤黄緑のウィンドウボタンあり
- ユーザーがリサイズ可能
- 最小サイズは `960 x 680`
- 構成は `Home / Trash / Settings` の3タブ
- Homeは `日時 > セッション > メモ` の階層表示
- 並びは日時降順
- メモカードはポストイット風UI
- タイトルは本文先頭 `10文字程度`
- 検索はキーワード一致のみ

## 10. セッション見出し
- セッション見出しは以下の形式で表示する
  - `2026/02/03 13:12のセッション`
- 内容は `セッション作成日時` のみとする

## 11. 保存方針
- 完全ローカル保存
- SQLite使用
- 5分ごとに未保存変更のみ内部自動保存
- 自動保存は、開いているセッションが存在する時だけ実行する
- 自動閉じる時間は `5 / 10 / 30 / 60分`
- 選択中・編集中は自動閉じない
- 自動閉じる時は保存してから非表示にする
- 自動シャットダウン機能は持たない

## 12. ゴミ箱
- 削除は即物理削除しない
- 30日間ゴミ箱保持
- 復元可能
- 完全削除可能
- 30日経過で自動削除

## 13. デザイン方針
- ライト前提
- 管理画面のみ将来ダーク対応の余地あり
- 全体の空気感は Apple純正メモ寄り
- 背景は白〜生成り寄り
- セッション色は実物付箋のような目に優しい色
- 選択色は青
- 警告色は柔らかい赤

## 14. アイコン方針
- アプリアイコンとメニューバーアイコンは共通モチーフにする
- 付箋1枚を抽象化した形にする
- 右下が軽くめくれている、または右上に折り返しがある形を採用する
- シンプルで識別しやすいことを優先する

- 表示時間は `900ms`

### 管理画面上の警告
- 文言として常時表示する
- セッション上限文言:
  - `これ以上セッションは開けません`
  - `展開中セッションは最大5件です`
  - `新しいセッションを開くには、既存のセッションを閉じてください`

  - `これ以上メモは表示できません`
  - `同時表示できるメモは最大15件です`
  - `新しいメモを表示するには、既存のメモを閉じてください`

## 20. 保存の扱い
- 保存は追加保存ではなく、現在本文の更新保存を意味する
- `Cmd + S` は選択対象の現在本文を更新保存する
- `Cmd + Enter` は更新保存して閉じる
- `Cmd + W` は計画から外す

## 21. Tauri の理解
### 概要
- UIはWeb技術で作る
- ただしブラウザアプリではなく、デスクトップアプリとして動かす
- Reactで見た目を作り、SQLiteやネイティブ機能はTauri経由で扱う

### 良い点
- Reactが使える
- 状態管理しやすい
- 比較的軽い
- ローカルアプリとして配布できる
- SQLiteやショートカット周りも比較的扱いやすい

### 注意点
- macOSの超細かいネイティブ挙動はSwift/AppKitほど自然ではないことがある
- メニューバー常駐や最前面制御は、純ネイティブより少し癖が出る可能性がある
- ただし現段階の難所はネイティブ感より `状態管理と操作設計` にあるため、MVPはTauriで進める方針で問題ない
