# Phase 3 骨組み計画書: セッション操作の成立

## 1. 対象フェーズ
- マスタープラン対応: `Phase 3: セッション操作の成立`
- 主目的: `1枚メモ前提` で成立した基礎操作を、`セッション` 単位の操作モデルへ拡張する

## 3. フェーズ目的
- 複数枚セッションを UI 上で扱えるようにする
- 右クリックメニュー経由のセッション選択を成立させる
- セッション単位の閉じる / 削除 / 前面化を成立させる
- 色スロット割り当てルールを UI と整合させる

---

## 4. 想定対象
- 複数枚セッション開始
- セッション色の表示
- セッション選択状態
- セッション一括移動
- セッション右クリックメニュー
- セッション単位の閉じる
- セッション単位の削除
- 最小メニューバー導線

---

## 5. 未確定論点
- セッション一括移動の実装粒度をどこまで Phase 3 に含めるか
- 右クリックメニューから入るセッション選択の視覚表現
- 複数枚セッション開始時の初期配置をどう見せるか
- セッション単位削除時の確認UIをどの粒度で出すか
- auto close をこのフェーズに含めるか、`last_active_at` を持つ Phase 4 に持ち越すか
- セッション一括移動を Gate 条件に含めるか、Phase 3 詳細計画で再判定するか

---

## 6. 想定 Issue 系統
- `S-`: セッション選択とメモ選択の状態分離
- `U-`: セッション色・選択リング・右クリックメニュー
- `D-`: セッション単位閉じる時の open 状態管理
- `K-`: セッション操作と Phase 2 の単体操作の整合

---

## 7. Gate 条件（骨組み）
- 複数枚セッションを生成して表示できる
- 右クリックメニューからセッション選択に入れる
- セッション選択時に視覚的区別がある
- セッション単位の閉じるが成立する
- セッション単位の削除が成立する
- 色スロットが開いているセッション数と整合する
- 最小メニューバーから `New 1-Note Session` と主要導線を開ける

---

## 8. フェーズ着手条件
- `Phase 2` が完了している
- 1枚メモの選択 / 編集 / 保存 / 閉じるが安定している
- 透過前面レイヤーの疎通確認を壊していない
