# Post it ロードマップ

## 1. 方針
本ロードマップは、`sticky` の初期MVPを最短で形にしつつ、後から拡張しやすい順序で進めるための計画である。  
優先順位は常に以下とする。

1. 思考の初速を落とさない
2. シンプルに成立させる
3. ローカル完結で仕上げる
4. 後から拡張できる構造にする

## 2. フェーズ全体像
### Phase 0: 仕様固定
目的:
- 今まで決めた内容を文書として固定する

成果物:
- 要件定義
- 画面一覧 / 状態遷移 / DBスキーマ案
- 操作一覧表
- 事前検討まとめ
- 疎通確認項目

完了条件:
- MVPの動作仕様に大きな迷いがない

### Phase 1: 技術疎通確認
目的:
- Tauri v2 + React + SQLite で詰まりそうな機能を小さく確認する

確認項目:
- グローバルショートカット
- 最前面表示
- メニューバー常駐
- SQLite保存
- 複数メモ描画
- リサイズ
- 管理画面ウィンドウ

成果物:
- 疎通確認メモ
- 採用可否判断

完了条件:
- MVP実装が技術的に成立すると判断できる

### Phase 2: UI骨格作成
目的:
- 実装前に画面構造とデザイン基盤を固定する

対象:
- 画面ワイヤー
- デザイントークン
- コンポーネント分割

成果物:
- 画面ワイヤー仕様
- design token 定義
- コンポーネント構成案

完了条件:
- 実装時にレイアウトで迷わない状態になる

### Phase 3: コア実装
目的:
- デスクトップメモ機能の最小成立

対象:
- 1枚セッション作成
- 複数枚セッション作成
- 最前面表示
- スロット配置
- 選択 / 編集 / 保存 / 削除
- セッション単位 / メモ単位の操作
- リサイズ

成果物:
- デスクトップで動く最小版 sticky

完了条件:
- メモを開いて、書いて、保存して、閉じられる

### Phase 4: 保存・管理画面実装
目的:
- 書いた内容を回収できるようにする

対象:
- SQLite保存
- 管理画面 Home
- Trash
- Settings
- セッション移動
- 検索
- 再表示導線

成果物:
- 管理画面付きMVP

完了条件:
- 保存済みメモを管理画面から確認・再表示できる

### Phase 5: 安定化と磨き込み
目的:
- 毎日使えるレベルまで違和感を減らす

対象:
- アニメーション調整
- 選択挙動の調整
- 上限警告の調整
- autosave / auto close の調整
- フォーカス遷移の調整
- メニューバー導線改善

成果物:
- 日常使用可能なMVP

完了条件:
- 自分で毎日使ってストレスが少ない

### Phase 6: 将来拡張
対象候補:
- アプリ/ウィンドウ文脈連動メモ
- iPhone版
- Windows版
- Android版
- 同期
- AI整理
- 高度検索

## 3. 直近の実行順
今すぐやる順番は以下とする。

1. 画面ワイヤー仕様を作る
2. デザイントークンをコードへ落とす
3. Tauri の疎通確認をする
4. デスクトップメモUIを最小実装する
5. SQLite保存をつなぐ
6. 管理画面を実装する

## 4. マイルストーン
### マイルストーン1
- グローバル起動キーで1枚セッションが開く
- メモを入力できる
- `Cmd + Enter` で保存して閉じる

### マイルストーン2
- 複数枚セッションが開く
- セッション選択 / メモ選択 / 編集の状態遷移が動く
- リサイズが動く

### マイルストーン3
- SQLite保存が動く
- autosave が動く
- auto close が動く

### マイルストーン4
- Home / Trash / Settings が動く
- メモ再表示ができる
- セッション移動ができる

### マイルストーン5
- 日常使用で大きな違和感が減っている

## 5. 注意点
- MVP段階ではクラウドや同期を入れない
- Markdown装飾は入れない
- 保存は更新保存として扱う
- autosave は開いているセッションが存在する時だけ動かす
- 自動シャットダウンは入れない
- `Cmd + W` は採用しない

## 6. 判断ポイント
Phase 1 の疎通確認で以下に大きな制約が出た場合は、Swift/AppKit の再検討を行う。

- 最前面表示
- メニューバー常駐
- グローバルショートカット
- フォーカス制御

## 7. 結論
現時点では、まず Tauri v2 + React + SQLite で MVP を成立させる方針が最も合理的である。  
本ロードマップに従えば、過剰な実装を避けつつ、思考キャプチャ体験の核を先に検証できる。
